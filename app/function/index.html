Controller Function Types Guide

Understand the different types of functions you can create in Kyte controllers and how to use them effectively.

Hooks - Extend Default Behavior

Hooks are lifecycle methods that allow you to inject custom logic at specific points in the request/response cycle without completely overriding the default CRUD operations.

hook_init

Runs when the controller is initialized, before any action is executed. Use this to set up initial state, load configuration, or perform authentication checks.

protected function hook_init() {
    // Example: Load application-specific settings
    $settings = new Model(AppSettings);
    $config = $settings->retrieve('app_id', $this->api->app->id);
    $this->appConfig = $config->getObjects()[0] ?? null;

    // Example: Set custom timezone
    date_default_timezone_set('America/New_York');
}
üí° Use Case: Initialize shared resources, load configuration, set timezone, or establish database connections that will be used across multiple actions.
hook_auth

Additional authentication logic that runs AFTER the default session validation passes. Use this to add extra authentication checks or load additional user context.

protected function hook_auth() {
    // Example: Check if user has specific permission
    if (!$this->user->hasPermission('access_controller')) {
        throw new \Exception('User does not have required permission');
    }

    // Example: Load additional user context
    $profile = new ModelObject(UserProfile);
    if ($profile->retrieve('user_id', $this->user->id)) {
        $this->userProfile = $profile;
    }

    // Example: Check IP whitelist for sensitive operations
    $allowedIPs = ['192.168.1.100', '10.0.0.50'];
    $clientIP = $_SERVER['REMOTE_ADDR'] ?? '';

    if (!in_array($clientIP, $allowedIPs)) {
        throw new \Exception('Access denied from this IP address');
    }

    // Example: Verify two-factor authentication
    if ($this->user->requires_2fa && !$this->verify2FA()) {
        throw new \Exception('Two-factor authentication required');
    }
}
üí° Use Case: Add role-based access control, check additional permissions, verify IP whitelists, enforce two-factor authentication, or load user-specific configuration that will be used throughout the request.
‚ÑπÔ∏è Note: This hook runs AFTER default authentication. The user session has already been validated. Throw an exception to deny access.
hook_prequery

Modify database query conditions before data is retrieved in GET, UPDATE, and DELETE operations. Add filters, joins, or custom conditions.

protected function hook_prequery($conditions) {
    // Example: Filter by user's organization
    if ($this->user && $this->user->organization) {
        $conditions[] = [
            'field' => 'organization',
            'value' => $this->user->organization,
            'operator' => '='
        ];
    }

    // Example: Only show active records
    $conditions[] = [
        'field' => 'status',
        'value' => 'active',
        'operator' => '='
    ];

    // Example: Add date range filter
    $conditions[] = [
        'field' => 'date_created',
        'value' => strtotime('-30 days'),
        'operator' => '>='
    ];

    return $conditions;
}
üí° Use Case: Implement row-level security, filter by tenant/organization, add soft delete conditions, or restrict data access based on user roles.
hook_preprocess

Transform and validate request data before it's saved to the database (called during CREATE and UPDATE operations). Perfect for data sanitization, validation, and computed fields.

protected function hook_preprocess($data) {
    // Example: Generate slug from title
    if (isset($data['title']) && empty($data['slug'])) {
        $data['slug'] = strtolower(str_replace(' ', '-', $data['title']));
        $data['slug'] = preg_replace('/[^a-z0-9-]/', '', $data['slug']);
    }

    // Example: Validate email format
    if (isset($data['email'])) {
        if (!filter_var($data['email'], FILTER_VALIDATE_EMAIL)) {
            $this->error('Invalid email format');
        }
    }

    // Example: Calculate total from line items
    if (isset($data['quantity']) && isset($data['price'])) {
        $data['total'] = $data['quantity'] * $data['price'];
    }

    // Example: Set default values
    if (!isset($data['status'])) {
        $data['status'] = 'pending';
    }

    // Example: Auto-assign to current user
    if (!isset($data['assigned_to'])) {
        $data['assigned_to'] = $this->user->id;
    }

    return $data;
}
üí° Use Case: Auto-generate slugs, validate input data, calculate computed fields, set default values, normalize data formats, or sanitize user input.
hook_response_data

Transform response data before sending it to the client. Add computed properties, format dates, hide sensitive fields, or enrich data with related records.

protected function hook_response_data($data) {
    // Example: Add computed property
    $data['full_name'] = trim($data['first_name'] . ' ' . $data['last_name']);

    // Example: Format timestamps
    if (isset($data['date_created'])) {
        $data['created_formatted'] = date('Y-m-d H:i:s', $data['date_created']);
        $data['created_relative'] = $this->timeAgo($data['date_created']);
    }

    // Example: Add status badge
    $statusColors = [
        'active' => 'success',
        'pending' => 'warning',
        'inactive' => 'danger'
    ];
    $data['status_badge'] = $statusColors[$data['status']] ?? 'secondary';

    // Example: Load related data
    if (isset($data['author'])) {
        $author = new ModelObject(User);
        if ($author->retrieve('id', $data['author'])) {
            $data['author_details'] = [
                'name' => $author->name,
                'avatar' => $author->avatar_url
            ];
        }
    }

    return $data;
}
üí° Use Case: Add computed fields, format dates for display, calculate aggregates, load related data not in the model definition, or transform data structure for frontend consumption.
hook_process_get_response

Process GET request responses specifically. Similar to hook_response_data but only runs for GET operations. Useful for adding view-specific data.

protected function hook_process_get_response($data) {
    // Example: Add permissions for current user
    $data['can_edit'] = $this->userCanEdit($data['id']);
    $data['can_delete'] = $this->userCanDelete($data['id']);

    // Example: Add view count
    $views = new Model(PageView);
    $viewCount = $views->count([
        ['field' => 'page_id', 'value' => $data['id']]
    ]);
    $data['view_count'] = $viewCount;

    // Example: Calculate time-sensitive data
    $data['is_expired'] = $data['expiry_date'] < time();
    $data['days_remaining'] = ceil(($data['expiry_date'] - time()) / 86400);

    return $data;
}
üí° Use Case: Add user-specific permissions, calculate view counts, add time-sensitive computed fields, or enrich GET responses with contextual data.
Overrides - Replace Default Operations

Overrides completely replace the default CRUD operations. Use these when you need full control over the entire operation logic, not just pre/post processing.

new

Override the default CREATE operation (POST requests). When this function exists, Kyte skips the default create logic entirely.

protected function new($data) {
    // Example: Custom validation
    if (empty($data['email']) || empty($data['password'])) {
        $this->error('Email and password are required');
    }

    // Example: Check for duplicates
    $existing = new ModelObject($this->modelName);
    if ($existing->retrieve('email', $data['email'])) {
        $this->error('User with this email already exists');
    }

    // Example: Hash password
    $data['password'] = password_hash($data['password'], PASSWORD_BCRYPT);

    // Example: Generate verification token
    $data['verification_token'] = bin2hex(random_bytes(32));
    $data['is_verified'] = 0;

    // Create the record
    $obj = new ModelObject($this->modelName);
    $obj->create($data, $this->user);

    // Example: Send welcome email
    $this->sendWelcomeEmail($obj->email, $obj->verification_token);

    // Example: Create related records
    $profile = new ModelObject(UserProfile);
    $profile->create([
        'user_id' => $obj->id,
        'theme' => 'default'
    ], $this->user);

    return $this->success([$obj->getAllParams()]);
}
üí° Use Case: Complex validation, creating multiple related records atomically, triggering external APIs, sending notifications, or implementing custom business logic during creation.
update

Override the default UPDATE operation (PUT requests). Complete control over how records are updated.

protected function update($data) {
    // Get the record to update
    $obj = new ModelObject($this->modelName);
    if (!$obj->retrieve($this->api->field, $this->api->value)) {
        $this->error('Record not found');
    }

    // Example: Permission check
    if ($obj->user_id !== $this->user->id && !$this->user->is_admin) {
        $this->error('You do not have permission to update this record');
    }

    // Example: Track changes for audit
    $changes = [];
    foreach ($data as $field => $newValue) {
        if ($obj->$field !== $newValue) {
            $changes[$field] = [
                'old' => $obj->$field,
                'new' => $newValue
            ];
        }
    }

    // Example: Prevent certain field updates
    unset($data['user_id']); // Don't allow changing owner
    unset($data['date_created']); // Don't allow changing creation date

    // Update the record
    $obj->save($data, $this->user);

    // Example: Log changes
    if (!empty($changes)) {
        $this->logAudit('update', $obj->id, $changes);
    }

    // Example: Invalidate cache
    $this->clearCache('user_' . $obj->id);

    return $this->success([$obj->getAllParams()]);
}
üí° Use Case: Permission checking, audit logging, preventing updates to certain fields, validating state transitions, or triggering workflows on specific changes.
get

Override the default READ operation (GET requests). Complete control over data retrieval and response format.

protected function get() {
    // Example: Custom query with joins
    $sql = "SELECT u.*, p.bio, p.avatar_url, COUNT(o.id) as order_count
            FROM User u
            LEFT JOIN UserProfile p ON p.user_id = u.id
            LEFT JOIN Order o ON o.user_id = u.id
            WHERE u.deleted = 0";

    if ($this->api->field && $this->api->value) {
        $sql .= " AND u." . DBI::escape_string($this->api->field) . " = '" . DBI::escape_string($this->api->value) . "'";
    }

    $sql .= " GROUP BY u.id ORDER BY u.date_created DESC";

    $result = DBI::query($sql);
    $data = [];

    while ($row = $result->fetch_assoc()) {
        // Example: Transform data
        $row['is_vip'] = $row['order_count'] >= 10;
        $row['member_since'] = date('Y-m-d', $row['date_created']);

        // Example: Add computed fields
        $row['full_name'] = $row['first_name'] . ' ' . $row['last_name'];

        $data[] = $row;
    }

    return $this->success($data, count($data));
}
üí° Use Case: Complex joins across multiple tables, custom SQL with aggregations, non-standard query logic, or returning data in a completely different structure.
delete

Override the default DELETE operation. Control deletion logic, implement hard deletes, or cascade deletions.

protected function delete() {
    // Get the record
    $obj = new ModelObject($this->modelName);
    if (!$obj->retrieve($this->api->field, $this->api->value)) {
        $this->error('Record not found');
    }

    // Example: Permission check
    if ($obj->user_id !== $this->user->id && !$this->user->is_admin) {
        $this->error('You do not have permission to delete this record');
    }

    // Example: Prevent deletion based on status
    if ($obj->status === 'active' || $obj->has_children) {
        $this->error('Cannot delete active records or records with dependencies');
    }

    // Example: Cascade delete related records
    $comments = new Model(Comment);
    $comments->deleteObjects([
        ['field' => 'post_id', 'value' => $obj->id]
    ]);

    // Example: Archive instead of delete
    $archive = new ModelObject(ArchivedPost);
    $archive->create($obj->getAllParams(), $this->user);

    // Example: Hard delete (not soft delete)
    $obj->purge($this->api->field, $this->api->value);

    // Example: Trigger cleanup
    $this->deleteAssociatedFiles($obj->id);
    $this->clearCache('post_' . $obj->id);

    return $this->success(['deleted' => true, 'id' => $obj->id]);
}
‚ö†Ô∏è Warning: By default, Kyte performs soft deletes (sets deleted=1). Override this function only if you need hard deletes or custom deletion logic.
Custom - Create Your Own Methods

Custom functions allow you to create endpoints for operations that don't fit the standard CRUD pattern. Perfect for business logic, workflows, calculations, and integrations.

custom (any method name)

Create any public method with any name. These are called via custom API endpoints and can perform any operation you need.

// Example: Send email notification
public function sendEmail($data) {
    $to = $data['to'] ?? '';
    $subject = $data['subject'] ?? '';
    $body = $data['body'] ?? '';

    if (empty($to) || empty($subject)) {
        $this->error('Email address and subject are required');
    }

    // Use Kyte's SES integration
    $ses = new \Kyte\Aws\SES($this->api->aws_key_id, $this->api->aws_secret);
    $result = $ses->sendEmail($to, $subject, $body);

    return $this->success(['sent' => true, 'message_id' => $result]);
}

// Example: Process payment
public function processPayment($data) {
    $amount = $data['amount'] ?? 0;
    $token = $data['stripe_token'] ?? '';

    // Validate
    if ($amount <= 0) {
        $this->error('Invalid amount');
    }

    // Process with Stripe
    $stripe = new \Stripe\StripeClient(KYTE_APP_ENV['STRIPE_SECRET_KEY']);
    $charge = $stripe->charges->create([
        'amount' => $amount * 100,
        'currency' => 'usd',
        'source' => $token,
    ]);

    // Record transaction
    $transaction = new ModelObject(Transaction);
    $transaction->create([
        'user_id' => $this->user->id,
        'amount' => $amount,
        'stripe_charge_id' => $charge->id,
        'status' => $charge->status
    ], $this->user);

    return $this->success(['transaction_id' => $transaction->id]);
}

// Example: Generate report
public function generateReport($data) {
    $startDate = $data['start_date'] ?? strtotime('-30 days');
    $endDate = $data['end_date'] ?? time();

    // Query data
    $orders = new Model(Order);
    $results = $orders->from('date_created', $startDate, $endDate, false, true);

    // Calculate metrics
    $total_revenue = 0;
    $total_orders = count($results);

    foreach ($results as $order) {
        $total_revenue += $order['total'];
    }

    $report = [
        'period' => [
            'start' => date('Y-m-d', $startDate),
            'end' => date('Y-m-d', $endDate)
        ],
        'metrics' => [
            'total_orders' => $total_orders,
            'total_revenue' => $total_revenue,
            'average_order_value' => $total_orders > 0 ? $total_revenue / $total_orders : 0
        ],
        'orders' => $results
    ];

    return $this->success($report);
}

// Example: Import data from CSV
public function importCSV($data) {
    $file = $_FILES['csv_file'] ?? null;

    if (!$file) {
        $this->error('No file uploaded');
    }

    $csv = array_map('str_getcsv', file($file['tmp_name']));
    $headers = array_shift($csv);

    $imported = 0;
    $errors = [];

    foreach ($csv as $index => $row) {
        $rowData = array_combine($headers, $row);

        try {
            $obj = new ModelObject($this->modelName);
            $obj->create($rowData, $this->user);
            $imported++;
        } catch (Exception $e) {
            $errors[] = "Row " . ($index + 2) . ": " . $e->getMessage();
        }
    }

    return $this->success([
        'imported' => $imported,
        'total' => count($csv),
        'errors' => $errors
    ]);
}
üí° Use Case: Send emails, process payments, generate reports, import/export data, trigger workflows, call external APIs, run batch operations, or any custom business logic.
üì° API Endpoint: Custom functions are accessible via POST /api/{ControllerName}/{functionName}. Example: POST /api/Order/processPayment
Best Practices
  • Use hooks when possible: Hooks are less intrusive and easier to maintain than overrides.
  • Return modified data: Hooks like hook_preprocess and hook_response_data must return the data array.
  • Error handling: Use $this->error('message') to return error responses properly.
  • Success responses: Use $this->success($data, $count) for consistent API responses.
  • Access control: Always check $this->user permissions before sensitive operations.
  • Use ModelObject for single records: new ModelObject(ModelName)
  • Use Model for collections: new Model(ModelName)
  • Test thoroughly: Functions run in production - test all edge cases.
  • Document your code: Add comments explaining complex logic for future maintainers.
  • Keep functions focused: Each function should do one thing well.
